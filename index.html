The self-hosted Dribbble Playbook alternative

Ever wanted to have your Dribbble shots automagically on a portfolio website with your custom domain? You've found the right project: Basketball. It's a self-hosted alternative to the paid Dribbble Playbook.

What is this?

Basketball is server running a custom profile with your Dribbble data and shots. It's relatively easy to set up if you have some basic coding/server knowledge and even runs in serverless environments thanks to FastAPI.

Installation

Here's what you need to get started:

- git
- nano
- Server (min. 1GB RAM / 5GB HDD/SSD) or Serverless enviroment (e.g. Vercel/DETA)
- preferably a domain

RECOMMENDED: Cloudflare SSL

The server or serverless enviroment you use has to support FastAPI and have Python 3.8+ support. Check if your server supports Python with python -v or python3 -v. To check if your serverless environment supports Python, consolidate your provider's website or documentation.

1. Download and configure

1. Clone the GitHub Repo into your desired folder with git clone -b main https://github.com/berrysauce/basketball.git and cd into your folder
2. Install all requirements with pip install -r requirements.txt (you may need to use pip3)
3. Create a .env with nano .env and paste the following text inside (for now, don't replace placeholders)

#DRIBBBLE_TOKEN = "BLANK"
DRIBBBLE_USERNAME = "YOUR_DRIBBBLE_USERNAME"
DRIBBBLE_CLIENT_ID = "YOUR_DRIBBBLE_CLIENT_ID"
DRIBBBLE_CLIENT_SECRET = "YOUR_DRIBBBLE_CLIENT_SECRET"
APP_HOST = "localhost"
APP_PORT = "8000"

If you have problems with binding to localhost:8000, change the values.
4. Save the .env and start the server with python main.py (you may need to use python3)

2.3 Install on Deta

1. Create an account on deta.sh
2. Install the Deta CLI like shown here
3. Log into your account from the CLI with deta login
4. Navigate to your installation's folder and enter deta new -n basketball --python in the CLI
5. Run deta deploy and wait for DETA to copy the code, install dependencies, and deploy your installation - you'll be given a URL to the installation

3 Configure API access

If you've installed Basketball on Vercel/DETA, you have to edit the .env variables differently!
on DETA: Edit .env locally and then push them with deta update -e .env and deta deploy
on Vercel: Edit the environmental variables in the project dashboard

1. Log in to your Dribbble account and visit this page to create your application
2. Enter a name and description of your choice, enter your installation's domain as the domain (https://yourdomain.com) and your installation's domain with /auth as the callback URL (https://yourdomain.com/auth) - then click create
3. Open the .env with nano .env again and copy/paste the Client ID, Client Secret and your Dribbble username
4. Restart your server with Ctrl+C and python main.py and visit the /auth page on your site (https://yourdomain.com/auth)
5. Allow your app to access your Dribbble data - you should be redirected to your installation which should now show a token to insert into the .env
6. Open the .env again, remove the # in front of the DRIBBBLE_TOKEN variable, and add the on /auth displayed token (replace the placeholder)
7. Restart your server again
